# LogMapping — 사용설명서

**오프라인 드라이브 파일 카탈로그.** 외장 하드디스크·SSD·USB 등 이동식 저장 매체의 파일 목록을 로컬에 보존하여, **매체를 연결하지 않은 상태에서도** 파일명·경로·확장자로 검색할 수 있는 Windows 단일 실행 도구입니다. Windows 드라이브뿐 아니라 **APFS·HFS+로 포맷된 macOS 드라이브**도 읽을 수 있으며, 각 드라이브는 드라이브 문자(`E:\`)가 아닌 **볼륨 일련번호**로 식별되어 같은 포트에 매체를 교체해도 서로 혼동되지 않습니다.

> 현재 버전: **BETA Ver-0.1** · 설치 불필요, 단일 실행 파일
>
> 본 문서는 배포되는 PDF 실행 안내서와 동일한 내용입니다.

## 1. 시작 조건 및 실행

### 1.1 시스템 요구사항

| 항목 | 요구사항 | 비고 |
|---|---|---|
| 운영체제 | Windows 10 이상 (64비트) | Windows 11 권장 |
| 웹 엔진 | Microsoft WebView2 런타임 | Windows 11 기본 내장. Windows 10은 Edge 설치 시 자동 포함 |
| 저장 공간 | 실행 파일 약 160 MB + 카탈로그 | 카탈로그 크기는 파일 수에 비례 |
| 네트워크 | 불필요 | 완전 오프라인 동작 |
| 관리자 권한 | **필요 (UAC 승인)** | 실행 시마다 표시. 물리 디스크(APFS 등) 스캔에 요구 |

### 1.2 최초 실행 절차

1. **`LogMapping.exe` 실행** — 더블클릭한다. 설치 절차는 없다. 최초 실행 시 내부 런타임 초기화로 2–3초 지연될 수 있으며 이후에는 즉시 구동된다.
2. **Windows SmartScreen 처리** — 인터넷에서 받은 파일에 최초 1회 경고가 표시되면 `[추가 정보] → [실행]`을 선택한다.
3. **관리자 권한 확인 (UAC) — 매 실행** — "이 앱이 디바이스를 변경하도록 허용하시겠어요?" 창에서 `[예]`를 선택한다. 물리 디스크 스캔에 필요한 정상 동작이다.
4. **WebView2 런타임 미설치 시** — 안내 창이 뜨면 <https://aka.ms/webview2> 에서 설치 후 재실행한다.

### 1.3 시작 화면 — 불러오기 / 새로 시작

실행하면 시작 화면이 나타난다. 프로그램 옆 `data/` 폴더에 저장된 카탈로그가 감지되면 **`[data/ 감지됨]`** 버튼으로 표시된다.

| 선택 | 동작 |
|---|---|
| 감지된 .hcat 불러오기 | 기존에 스캔한 드라이브 전체를 그대로 이어서 사용 |
| .hcat 파일 열기 | 다른 위치의 카탈로그 파일을 직접 선택해 열기 |
| 새 카탈로그 시작 | 빈 상태로 시작 — 새로 스캔. 기존 파일은 자동 백업(`.bak`)으로 보존 |

## 2. 드라이브 카탈로그 생성

드라이브를 스캔하면 파일 목록이 로컬 카탈로그에 저장된다. 이후 해당 드라이브를 분리한 상태에서도 목록 조회·검색이 가능하다.

### 2.1 스캔 절차

1. **드라이브 연결** — 스캔할 드라이브를 연결하고 운영체제가 인식하는지 확인한다.
2. **`[+ 드라이브 추가]` → 드라이브 선택** — 연결된 드라이브 목록에서 대상을 고른다. 폴더를 직접 선택할 수도 있다.
3. **번호 · 이름 · 정보 입력** — 드라이브 번호와 이름을 입력한다. 색상, 구매일·보증일, 건강 상태·메모를 함께 지정할 수 있다.
4. **`[스캔 시작]`** — 파일 수에 따라 수 초~수 분 소요된다. 스캔 중 드라이브를 분리하지 않는다.
5. **완료** — 사이드바에 드라이브가 추가되면 완료된 것이다.

### 2.2 Windows · macOS 드라이브

Windows 드라이브(NTFS·exFAT 등)와 **macOS 드라이브(APFS·HFS+)**를 모두 스캔할 수 있다. 맥 드라이브는 Windows 탐색기에 보이지 않아도 목록에 **`[APFS]/[HFS+] Mac 드라이브`**로 나타난다. 각 드라이브는 드라이브 문자가 아니라 **볼륨 일련번호**로 식별되므로, 같은 USB 포트에 다른 하드를 바꿔 꽂아도 서로 다른 항목으로 관리되어 덮어써지지 않는다.

### 2.3 재스캔

드라이브 정보 탭의 **`[재스캔]`**은 해당 드라이브를 다시 읽어 목록을 최신화한다. 재스캔 시 프로그램은 **연결된 드라이브가 원본과 같은 볼륨인지 확인**한다. 원본 드라이브가 연결돼 있지 않거나 같은 위치에 다른 드라이브가 연결돼 있으면 경고 후 중단하여, 엉뚱한 드라이브가 기존 기록을 덮어쓰지 않도록 한다. 드라이브 문자가 바뀌었어도 올바른 드라이브를 찾아 재스캔한다.

### 2.4 드라이브 정보 · 건강

드라이브 정보 탭에서 이름·번호·메모·태그와 함께 **구매일, 보증 만료일, 건강 상태(정상/주의/위험), 건강 메모**를 기록·수정할 수 있다. 수정 내용은 즉시 저장되어 다음 실행에도 유지된다. 보증 만료일이 지나면 붉게 표시된다.

## 3. 검색 및 탐색

### 3.1 키워드 검색

상단 검색창에 파일명·폴더명·경로의 일부를 입력하고 Enter를 누르면 등록된 전체 드라이브에서 부분 일치 검색이 실행된다. 확장자 검색(`.CR3` `.mov` `.pdf` 등)도 지원한다.

### 3.2 분류 · 색상 태그 필터

| 필터 | 설명 |
|---|---|
| 분류(카테고리) | 영상 · 사진 · 음악 · 문서 · 디자인 · 기타 · 폴더 로 걸러 보기 |
| 색상 태그 | 항목에 색상 태그를 지정하고, 태그 칩으로 해당 항목만 필터 |
| 드라이브 범위 | 사이드바에서 특정 드라이브 선택 시 해당 드라이브만 대상 |

### 3.3 경로 복사 · 보기 방식

항목을 마우스 오른쪽 버튼으로 클릭하면 **`[경로 복사]`**(전체 경로를 클립보드로)와 색상 태그 지정 메뉴가 나타난다. 빈 공간 우클릭으로 보기 방식(큰/보통/작은/자세히)을 바꿀 수 있다. '자세히'는 크기·확장자·수정일을 함께 표시한다.

## 4. 용량 표시

### 4.1 총 · 사용 · 여유

드라이브 정보 탭은 **총 용량 · 사용 중 · 여유 공간**과 사용률을 표시한다. 맥(APFS/HFS+) 드라이브는 볼륨 헤더에서 실제 용량을 읽으며, 총량과 사용량을 같은 기준으로 계산하여 여유 공간이 항상 올바르게(음수 없이) 표시된다.

### 4.2 드라이브 건강 정보

건강 상태·구매일·보증 만료일·메모가 함께 표시된다. 오래 스캔하지 않은 드라이브는 재스캔 권장 안내가 나타난다.

---

## 5. 중복 파일 탐지

여러 드라이브에 걸쳐 **이름과 크기가 같은** 파일을 찾아 중복 저장 여부를 파악한다.

1. **`[중복 파일]` 탭 선택** — 상단 탭에서 중복 탐지로 이동한다.
2. **결과 확인** — 동일 파일이 그룹 단위로 표시되며, 각 항목에 드라이브 색상 배지가 붙어 분포를 파악할 수 있다.

LogMapping은 파일을 직접 삭제·이동하지 않는다. 정리가 필요하면 `[경로 복사]` 후 운영체제 파일 탐색기에서 수동으로 처리한다.

## 6. 데이터 내보내기

### 6.1 CSV

전체 카탈로그를 CSV로 내보낸다. 드라이브번호·이름·경로·파일명·크기·분류·수정일 열을 가지며 Excel 등에서 열 수 있다. (한글 정상 표시를 위한 UTF-8 BOM 포함)

### 6.2 오프라인 HTML 뷰어

LogMapping이 없는 환경에서도 열 수 있는 독립형 HTML을 생성한다. 브라우저로 바로 열며 검색·폴더 탐색을 지원하고, 네트워크 없이 동작하는 열람 전용 형태다. 항목이 매우 많으면 드라이브별로 분할 생성된다.

## 7. 문제 해결

### 7.1 일반 오류 대처

| 증상 | 원인 | 대처 |
|---|---|---|
| 실행 직후 경고창 | Windows SmartScreen | `[추가 정보] → [실행]`. 최초 1회 |
| 실행마다 권한 창 | 디스크 스캔용 관리자 권한 (정상) | `[예]` 선택 필수 |
| WebView2 오류창 | 런타임 미설치 | <https://aka.ms/webview2> 설치 후 재실행 |
| 재스캔 시 "드라이브를 찾을 수 없습니다" | 원본 드라이브 미연결 / 다른 드라이브가 같은 위치 | 원본 드라이브를 연결한 뒤 다시 시도 (오스캔 방지 정상 동작) |
| 시작 시 드라이브 안 보임 | 카탈로그 미로드 | 시작 화면에서 `[data/ 감지됨]` 버튼으로 불러오기 |
| 첫 실행 2–3초 지연 | 런타임 초기화 (정상) | 대기. 이후 즉시 구동 |

### 7.2 자주 묻는 질문

**Q. 다른 컴퓨터에서도 쓸 수 있나?**
가능하다. `LogMapping.exe`와 `data/` 폴더를 함께 복사하면 동일한 카탈로그를 그대로 사용할 수 있다.

**Q. 드라이브를 연결하지 않아도 검색되나?**
가능하다. 카탈로그는 로컬에 저장되므로 드라이브 없이도 목록·검색이 동작한다.

**Q. 같은 USB 포트에 하드를 바꿔 꽂으면 섞이나?**
섞이지 않는다. 드라이브를 볼륨 일련번호로 식별하므로 서로 다른 항목으로 관리된다.

**Q. 맥에서 쓰던 하드도 읽나?**
읽는다. APFS·HFS+ 포맷 드라이브를 Windows에서 스캔할 수 있다.

**Q. 정보를 수정했는데 다음에 유지되나?**
유지된다. 이름·번호·건강 정보 수정은 즉시 저장된다.

## 빌드 (개발자용)

- **요구**: .NET 8 SDK
- **명령** (리포 루트에서):
  ```
  dotnet publish LogMapping/LogMapping.csproj -c Release -r win-x64 --self-contained true /p:PublishSingleFile=true /p:IncludeNativeLibrariesForSelfExtract=true -o publish_out
  ```
- 결과: `publish_out/LogMapping.exe` (self-contained 단일 파일, .NET 런타임 번들 포함).
- Visual Studio 2022로 열려면: `LogMapping/LogMapping.sln`

**기술 스택**: C# / .NET 8 (WPF) · WebView2 · SQLite(Microsoft.Data.Sqlite) · DiscUtils(APFS/HFS+ 읽기)
